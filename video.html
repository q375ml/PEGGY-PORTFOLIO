專案項目 Project Details

描述 Description－

2020滾動式數位轉型觀光振興培訓教育專案與政府合作數位轉型，主要培訓人員為旅行業、觀光旅館業、旅館業及觀光遊業之從業人員，因為疫情的關係從原本的實體課程改成線上教學課程，因此產生了使用動機，加強相關重業人員進修藉此完成課堂專案和時數，不僅提升技能和領取相關證照也搭上數位轉型和符合市場需求，原本的通訊系統應用層面更為擴大進而提升整體的操作體驗。

本專案以使用者觀點切入，以使用者研究，從情境需求作為第一階段規劃。

第一階段的目標 First stage target－

提供 B2B 通訊系統應用，使用視訊線上教學平台，提供給觀光發展協會(管理者)、領隊或導遊執照者(老師)、旅館業及觀光遊樂業之重業人員（學生），前後台使用，滿足課堂學系以及自主進修情境。

情境 User story－

  1. 身為一個學生，我要藉由這次疫情數位轉型加強進修，因此我可以提升技能和拿到證照符合之後觀光市場所需。
  2. 身為一個老師，我要協助觀光發展協會數位轉型教學內容，因此我可以協助重業人員拿到相關證照。
  3. 身為一個管理者，我要藉由後台管理的狀況，因此我可以掌握老師與學生的上課狀況，和課程管理。

服務流程圖 Flow Chart

  • 服務流程圖繪製的元素有：
    * 用戶的目標
    * 為了完成目標，必要經歷的關鍵任務
    * 關鍵任務如何與利害關係人合作來完成

    上圖是用情境作為範例

    服務流程圖展示用戶如何達成目標，以及哪些角色參與了任務。
  • 角色目標：
    1. 學生(相關旅遊重業人員)
      學生接收管理者課程通知
      學生接收老師教學內容和新知
      學生使用線上教學系統上課
      學生完成專案和上課時數
      學生成功領取證照

    2. 老師（領隊或導遊執照）
      老師接收管理者上課通知（課堂主持人）
      老師準備課程簡報教學
      老師可檢視學生學習時數和課堂狀況

    3. 管理者（觀光發展協會）
      負責提供老師、學生帳號密碼
      提供課程資訊
      建立課程資訊給學員報名，新增、編輯
      安排老師上課
      聯絡學生和老師
      可管理課程內容和學生名單
      協助學生帳號批次通過審核
  • 定義任務：
    1. 學生(相關旅遊重業人員)
      登記課程 → 審核通過 → Email通知 → 上課 → 完成課程專案和時數 → 拿到相關證照

    2. 老師（領隊或導遊執照）
      課堂主持人 → Email通知 → 上課 → 檢視學生學習進度 → 協助學生拿到證照

    3. 管理者（觀光發展協會）
      安排老師、聯絡學員 → 建立帳密(老師/學生) → 提供課程資訊 → 管理課程、學生資料 → 協助老師學生順利上課
  • 使用可用性三原則確認需求，達成共識：

    將用戶研究的結果與用戶的任務分析轉換為服務流程圖

    簡單的功能跟開發時間越少得越往前排。

設計 UI Design
  • 前台：
    經過上面Wireframes確認功能沒問題後，開始發想視覺，主要風格清新、乾淨、簡潔、舒服、穩重、科技，在過程中定義設計樣式，元素和顏色。使用Adobe XD 設計 UI 外觀，幫助使用者在操作介面上能直覺達到目標。
    考慮到工程師事後需要 UI 元件庫，也一併整理 UI system。
  • 後台：
    參考 element UI https://element.eleme.io/#/zh-CN ，在開發時間不足的情況下，會使用有open source UI 元件庫，直接在 VScode 進行前端畫面開發。 流程圖畫完確認功能沒問題，進 VScode 使用 HTML、SCSS、簡易JS，製作動態，協助工程師後續製作。
